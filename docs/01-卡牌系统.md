# 卡牌系统

> 本文档定义游戏的卡牌体系：四大类卡牌、消耗规则、属性系统、变招/追击升级。
>
> 相关文档：[00-总览.md §2](./00-总览.md) 资源四维度 | [02-怪物与战斗.md](./02-怪物与战斗.md) 战斗公式 | [03-事件与驿站.md §6](./03-事件与驿站.md) 卡牌升级事件

## 1. 卡牌总览

### 1.1 通用规则

- 每回合固定抽 5 张手牌
- 每回合 3 点行动力
- 出牌消耗对应资源（HP/MP/金币/技能/影响力等）
- 回合结束弃置所有剩余手牌
- 战斗胜利可获得新卡牌或卡牌升级机会

### 1.2 卡牌实例结构

`gameState.deck` 中每个元素的运行时结构：

```js
{
  instanceId: 'card_001',           // 实例 ID（支持多张同名）
  id: 'atk1',                       // 引用 CARDS.attackCards 配置
  attributes: ['achievement'],      // 冗余存储卡牌属性（便于读取）
  upgrades: []                      // 已挂载的事件奖励标签
                                    //  - pursuit_* : 战斗中自动触发
                                    //  - variation_* : 战斗中手动激活(消耗 VP,每场 3 次上限)
}
```

### 1.3 卡牌四大类

- **攻击卡**（5 张）：造成伤害
- **防御卡**（5 张）：叠加护盾
- **功能卡**（5 张）：特殊机制
- **委托卡**（5 张）：消耗 HP+MP，获取资源

## 2. 攻击卡（冷兵器命名 + 属性系统）

> 全部 5 张攻击卡已重构为**冷兵器风格命名**，除 `战斧猛劈`（高阶卡）保留 HP 消耗外，其余均改为 **MP 消耗**。每张卡分配"成就感/影响力"单或双属性。

| ID | 名称 | 消耗 | 伤害 | 属性 | 描述 |
|----|------|------|------|------|------|
| atk1 | **英勇冲锋** | MP 5 | 15 | 成就感 | 骑枪直刺，以气势力压对手 |
| atk2 | **长剑斩击** | MP 8 | 22 | 影响力 | 双手握剑，全力劈砍 |
| atk3 | **双手重击** | MP 10 | 30 | 成就感+影响力 | 沉肩坠肘，重劈而下 |
| atk4 | **战斧猛劈** | **HP 15** | 40 | 影响力 | 战斧高举，蓄力劈下（**高阶卡，唯一保留 HP 消耗**） |
| atk5 | **骑枪冲刺** | MP 7 | 18 | 成就感+影响力 | 策马挺枪，贯穿敌阵 |

> **属性匹配规则**：卡牌可含 1~2 个属性；与怪物交互时**取最高伤害倍率**（不叠加）。详见 [02-怪物与战斗.md §2.1](./02-怪物与战斗.md)。

> **变招 / 追击 升级**：5 张攻击卡可作为事件奖励的挂载目标，玩家通过卡牌升级事件永久添加变招 / 追击标签。
> - **追击** 仍为自动触发（满足条件即生效），纯增益、无代价。
> - **变招** 改为「战斗内手动选择」，激活时伤害 ×0.7 并附加特殊效果，需消耗 1 变招点（VP），每场战斗上限 3 次，仅战斗节点可用。
> 详见 [08-变招.md](./08-变招.md) 完整机制说明 与 [03-事件与驿站.md §6](./03-事件与驿站.md) 升级事件池。

## 6. 属性系统

### 6.1 双属性定义

- **成就感（achievement）**：以理服人、以势动人。代表沟通说服型攻击。
- **影响力（authority）**：以力压人、强制执行。代表压制型攻击。

### 6.2 卡牌属性分布

5 张攻击卡按"单/双属性"混合分配（见 §2 表格）：

- **单属性卡**（atk1、atk2、atk4）：主修一条路线，养成为某一属性流派的核心牌。对拥有该属性弱点的怪物伤害极高，但被抗性怪物克制时几乎打不出伤害。
- **双属性卡**（atk3、atk5）：综合型，灵活但缺乏极端倍率。适合应对属性未知的复杂战斗。

### 6.3 怪物属性交互（详细见 02）

普通怪：**无属性交互**，攻击牌按基础伤害结算。

精英怪 / BOSS：每个独立配置
- **弱点属性（weak）**：该属性牌伤害倍率 +200%
- **抗性属性（resist）**：该属性牌伤害倍率 -70%
- **普通属性（normal）**：该属性牌伤害倍率 100%
- **终结属性（finisherAttribute）**：怪物 HP 跌至阈值后，仅含此属性的牌能造成致命一击
- **终结阈值（finisherThreshold）**：触发终结态的 HP 比例（默认 20%）

**终结态机制**：
- HP / maxHP ≤ finisherThreshold → 进入终结态
- 含终结属性的牌 → 正常结算伤害，享受属性加伤
- 不含终结属性的牌 → **伤害强制置 0**（无法扣血，但非致命段可正常结算）
- 变招 / 追击追加的伤害也必须含终结属性，否则同样置 0
- 终结态被含终结属性的牌击杀 → 触发**"完美终结"**奖励（金币 / 经验 / 影响力 +20%）

**破局机制**（防卡死）：
- **商店**出售"属性印记"消耗品：临时为 1 张攻击牌添加终结属性（300 金币，1 次战斗有效）
- **驿站**提供"重新洗牌"：消耗 50 MP 重抽 5 张手牌

### 6.4 变招/追击升级（事件奖励）

通过卡牌升级事件，玩家可以为攻击牌**永久挂载**变招 / 追击标签，跨战斗、跨层保持。

**变招类（variation）**：战斗内**手动激活**，消耗 1 VP（变招点），每场战斗上限 3 次，仅战斗节点可用。

| 标签 ID | 名称 | 变招形态效果（伤害 ×0.7 + 特殊） | 适用场景 |
|---------|------|----------------------------------|----------|
| `variation_pierce` | 变招·穿刺 | 无视护盾 | 敌人有高护盾时 |
| `variation_chain` | 变招·连斩 | 追加同卡基础伤害 50% 段（独立结算） | 单回合高爆发 |
| `variation_cleave` | 变招·裂地 | 改为群攻所有敌人 | 多目标战斗 |
| `variation_execute` | 变招·终结 | 终结态下 ×2（净 ×1.4） | 残血收割 |
| `variation_momentum` | 变招·蓄势 | 本次 HP / MP 消耗减半（下限 1） | 高消耗卡 |

**追击类（pursuit）**：战斗中**自动触发**，满足条件即生效，纯增益、无代价。

| 标签 ID | 名称 | 触发器 | 效果 |
|---------|------|--------|------|
| `pursuit_strike` | 追击·连携 | `dupe_2` | 手牌同名 ≥2 张（不含本张），打出追加 50% 伤害 |
| `pursuit_bleed` | 追击·流血 | `dupe_2` | 手牌同名 ≥2 张，追加 50% + 流血 3 回合 |
| `pursuit_armor_break` | 追击·破甲 | `dupe_2` | 手牌同名 ≥2 张，追加 50% + 破甲 |
| `pursuit_echo` | 追击·回响 | `per_use` | 本张牌结算后免费追加 1 次（不消耗行动点） |

> **变招点（VP）**：每场战斗开局 3 点，全局共享。`gameState.variationPoints` 在 `startBattle()` 重置；`saveGame` 不持久化（每场重置）。
> **节点限制**：变招激活 UI 仅在战斗节点显示（`inBattle === true`），事件 / 驿站 / 商店不可变招。

**完整机制与 5 变招 + 4 追击详细说明** 详见 [08-变招.md](./08-变招.md)。
**事件池与获取方式** 详见 [03-事件与驿站.md §6](./03-事件与驿站.md)。

## 7. 卡牌数据配置示例

```js
const CARDS = {
  attackCards: [
    {
      id: 'atk1',
      name: '英勇冲锋',
      type: 'attack',
      cost: { type: 'mp', value: 5 },
      damage: 15,
      attributes: ['achievement'],
      description: '骑枪直刺，以气势力压对手'
    },
    {
      id: 'atk4',
      name: '战斧猛劈',
      type: 'attack',
      cost: { type: 'hp', value: 15 },  // 唯一保留 HP 消耗的高阶卡
      damage: 40,
      attributes: ['authority'],
      description: '战斧高举，蓄力劈下'
    }
    // ... 其他 3 张
  ],
  // defenseCards, functionCards, workCards 略
};
```

> **注意**：升级标签不存储在 `CARDS.attackCards` 配置中，而是存储在 `gameState.deck` 中每张卡牌实例的 `upgrades` 数组中（运行时状态）。
