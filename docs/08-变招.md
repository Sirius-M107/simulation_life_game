# 变招 战术选择体系

> 本文档定义游戏的"变招"机制:玩家通过升级事件为卡牌**永久挂载**变招标签,但**激活方式为战斗内手动选择**,需消耗变招点(VP),每场战斗上限 3 次。

## 0. 设计缘起

### 0.1 旧版范围(2026-07-10)

01 §6.4 + 03 §6 定义的"变招/追击"是**永久挂载 + 自动触发**机制:

| 维度 | 旧设计 |
|------|--------|
| 触发方式 | `usage_2` / `dupe_2` / `hp_le_20` / `per_use` 四种触发器自动判定 |
| 次数上限 | 无(每次满足条件都触发) |
| 节点限制 | 无(战斗中始终生效) |
| 效果性质 | 纯增益(+50% / +100% 伤害,无视护盾 等) |
| 代价 | 无 |
| 标签数量 | 7 个(4 变招 + 3 追击) |

### 0.2 用户决策(2026-07-11)

> "变招需要较大限制使用次数,并且只在招式施放在特定的节点上才能选择变招,变招会减轻此招式本来的威力。"

⇒ **本轮重构**:
1. 变招激活从「自动触发」改为「战斗内手动选择」
2. 每场战斗 3 次上限(全局共享变招点 VP 池)
3. 仅战斗节点允许变招(事件 / 驿站 / 商店不可变招)
4. 变招形态下伤害统一 ×0.7

⇒ **同步扩展**:
- 变招目标沿用 2026-07-10 的「动作牌」扩展(attack / defense / function 三类)
- 变招标签数量从 4 个扩展到 5 个,新增 `variation_momentum`(变招·蓄势)

### 0.3 新设计目标

1. **战术性**:玩家需要判断"这场战斗 3 个 VP 用在哪个回合、哪张牌"
2. **权衡性**:变招形态伤害更低,玩家要权衡"伤害换机制"是否划算
3. **节点绑定**:让变招成为"战斗专属"机制,与驿站强化、事件奖励形成三轨差异化
4. **可叠加**:同一张卡可挂多个变招 / 追击标签,变招 + 追击可同时生效

## 1. 体系总览

### 1.1 双轨制:变招 vs 追击

| 维度 | 变招(variation) | 追击(pursuit) |
|------|------------------|---------------|
| 配色 | 红色 `#c0504d` | 蓝色 `#4a7a8a` |
| 数量 | 5 | 4 |
| 激活方式 | **战斗内手动选择**(消耗 1 VP) | 自动触发(满足条件即生效) |
| 次数上限 | **每场 3 次**(VP 池全局共享) | 无 |
| 节点限制 | **仅战斗节点** | 无(战斗外不影响,因为不出牌) |
| 代价 | **伤害 ×0.7** | 无(纯增益) |
| 标签 ID 前缀 | `variation_*` | `pursuit_*` |

> **关键差异**:变招 = "花资源换机制",追击 = "白嫖的机制奖励"。这种双轨让玩家在构筑时有清晰的选择空间。

### 1.2 核心抽象:变招点(VP, Variation Points)

- **每场战斗开局**: `variationPoints = 3`(全局共享池)
- **每次变招激活**: `variationPoints -= 1`
- **VP 上限**:固定 3(本设计不引入增加上限的系统)
- **跨战斗**:每场战斗重置为 3
- **存档**:变招点不持久化(每场战斗重置即可)

### 1.3 标签清单(9 个 = 5 变招 + 4 追击)

| 大类 | ID | 名称 |
|------|----|------|
| 变招 | `variation_pierce` | 变招·穿刺 |
| 变招 | `variation_chain` | 变招·连斩 |
| 变招 | `variation_cleave` | 变招·裂地 |
| 变招 | `variation_execute` | 变招·终结 |
| 变招 | `variation_momentum` | 变招·蓄势 |
| 追击 | `pursuit_strike` | 追击·连携 |
| 追击 | `pursuit_bleed` | 追击·流血 |
| 追击 | `pursuit_armor_break` | 追击·破甲 |
| 追击 | `pursuit_echo` | 追击·回响 |

### 1.4 触发器语义变更

| 旧触发器 | 新行为 |
|----------|--------|
| `usage_2` (变招) | **废弃**——变招改为手动,不再依赖使用次数 |
| `hp_le_20` (变招) | **废弃**——`variation_execute` 改为手动且条件触发(终结态下 ×2) |
| `per_use` (变招) | **废弃**——不再有「每次都生效」的变招 |
| `dupe_2` (追击) | **保留**——3 个追击仍按手牌同名 ≥2 触发 |
| `per_use` (追击·回响) | **保留**——echo 仍是每次使用都生效 |

## 2. 变招点(VP)机制

### 2.1 初始化与重置

```js
// script.js startBattle() 中
gameState.variationPoints = 3;  // 每场战斗开局初始化
```

- `startBattle(enemy)` 是唯一初始化点(普通 / 精英 / BOSS 三类战斗节点都触发)
- `endBattle()` 不显式清零(下次 startBattle 会重置)
- 玩家跨层、跨战斗时 VP 自动重置

### 2.2 战斗 UI 显示

战斗场景顶部资源栏(在 HP / MP / 金币旁)新增:

```
[ HP 100 ] [ MP 80 ] [ 💰 5,000 ] [ ⚡ VP 3/3 ]
                                            └── 当前 VP / 上限
```

- VP = 0 时,变招按钮灰显且 tooltip 显示 "VP 已耗尽"
- VP ≥ 1 时,变招按钮高亮可用

### 2.3 消耗规则

每次玩家在战斗中选择「变招形态」出牌:

```js
gameState.variationPoints -= 1;
```

- 同张牌可多次变招(每次独立扣 1 VP)
- VP 跨回合不重置(整场战斗统一上限)
- 战斗中不允许 VP 回充(本设计不引入回充道具)

### 2.4 持久化

- `variationPoints` **不写入存档**(saveGame 不持久化)
- 理由:VP 是"战斗内短期资源",读档时刚好在战斗中间的状态会自然重置到当前战斗的初始值 3
- 实际实现:`saveGame` 序列化时不包含 `variationPoints`,读档时由 `startBattle` 重新初始化

## 3. 变招选择 UI(战斗内)

### 3.1 触发条件

玩家在战斗中出牌时,若**同时满足**以下条件,显示「变招形态」选项:

1. `gameState.inBattle === true`(在战斗中)
2. `card.upgrades` 数组中至少含一个 `variation_*` 标签
3. `gameState.variationPoints > 0`
4. 卡牌类型为 `attack` / `defense` / `function`(委托卡不参与,见 §9.4)

非战斗场景(事件 / 驿站 / 商店)即使卡牌有变招标签,**变招按钮也不显示**——这是节点限制的核心实现。

### 3.2 UI 形态

卡牌下方(或卡牌详情面板)增加一个 toggle 按钮:

```
┌────────────────┐
│ 英勇冲锋       │
│ ⚔️ 攻击 15 伤害  │
│                │
│ 骑枪直刺...     │
│                │
│ ┌────────────┐ │
│ │⚡ 变招 -1 VP│ │  ←  点击切换"变招形态"
│ └────────────┘ │
└────────────────┘
```

按钮状态:
- **可点击**(`VP > 0` 且有变招标签):高亮,点击切换变招形态
- **不可点击**(`VP = 0` 或无变招标签):灰显,tooltip 提示原因

变招形态下,卡牌显示强化效果预览(类似「变招·穿刺 → 伤害 ×0.7 + 无视护盾」)。

### 3.3 节点限制生效

非战斗场景(`inBattle === false`)的 UI 行为:

| 场景 | 卡牌可见? | 变招按钮可见? | 备注 |
|------|-----------|---------------|------|
| 普通战斗 | ✓ | ✓(若 VP > 0) | 主场景 |
| 精英战斗 | ✓ | ✓(若 VP > 0) | 同上 |
| BOSS 战斗 | ✓ | ✓(若 VP > 0) | 同上 |
| 事件节点 | ✓ | ✗ | 节点限制 |
| 驿站 | ✓ | ✗ | 节点限制 |
| 商店 | ✓ | ✗ | 节点限制 |

### 3.4 决策流程图

```
玩家点击手牌中的卡牌
  ↓
[卡牌详情面板打开]
  ├─ 检查 card.upgrades 含 variation_*?
  │   └─ 否 → 仅显示「普通形态」
  │
  ├─ 检查 inBattle && variationPoints > 0?
  │   └─ 否 → 显示「变招形态(不可用)」+ tooltip 原因
  │
  └─ 显示「变招形态」按钮(可点击)
        ↓ 玩家点击切换
      [出牌确认]
        ├─ 普通形态 → applyCardNormal(card)
        └─ 变招形态 → applyCardVariation(card)
              ├─ gameState.variationPoints -= 1
              ├─ 标记本次为变招形态(影响阶段 2.5)
              └─ 写入战斗日志: "⚡ [卡名] 启动变招 [变招名]! 消耗 1 VP"
```

## 4. 5 个变招类标签(新)

每个变招标签的字段定义:

```js
{
  id: 'variation_pierce',
  name: '变招·穿刺',
  kind: 'variation',
  effect: '伤害 ×0.7,无视护盾',
  color: '#c0504d',
  apply: ctx => ({ damageMul: 0.7, ignoreShield: true })
}
```

> 注意:**不再有 `trigger` 字段**(手动激活后无条件应用 ×0.7 + 特殊效果)。

### 4.1 `variation_pierce` 变招·穿刺

- **普通形态**:原伤害
- **变招形态**:伤害 ×0.7, **无视护盾**(阶段 8 直接扣 HP,不扣 shield)
- **适用场景**:敌人有高护盾(诅咒石像鬼 反射护盾等)
- **取舍**:伤害减 30%,换来"必中 HP"——对盾高的怪物收益显著

### 4.2 `variation_chain` 变招·连斩

- **普通形态**:原伤害
- **变招形态**:伤害 ×0.7, **追加一次同卡基础伤害 50% 的独立段**(独立走属性倍率与暴击)
- **适用场景**:玩家想单回合高爆发(对 BOSS 终结前冲伤害)
- **取舍**:主段伤害减 30%,但附加段不受 ×0.7 影响,可享受属性倍率与暴击
- **净伤害估算**:假设 attributeMul = 1,isCrit = false
  - 主段 = card.damage × 0.7
  - 追加段 = card.damage × 0.5 × attributeMul
  - 总和 = card.damage × (0.7 + 0.5 × attrMul)
  - 若 attrMul = 1:总 = 1.2 × card.damage(高于普通)
  - 若 attrMul = 2(弱点):总 = 1.7 × card.damage(显著高于普通)
- **设计意图**:对高属性倍率的精英 / BOSS,变招·连斩的"追加段"是真正的杀手锏

### 4.3 `variation_cleave` 变招·裂地

- **普通形态**:原伤害(单目标)
- **变招形态**:伤害 ×0.7, **改为群攻所有敌人**(若场上仅 1 个敌人,本次额外附加 50% 溅射伤害作为"空气波")
- **适用场景**:多敌人战斗(未来 PVE 内容扩展)、特定 BOSS 的多阶段
- **取舍**:单目标伤害减 30% + 仅一个敌人时多打 50% 溅射(也低于普通),价值在多目标场景
- **当前实现限制**:本 demo 大部分战斗是单目标,实际收益有限;作为未来多敌人扩展的伏笔

### 4.4 `variation_execute` 变招·终结

- **普通形态**:原伤害
- **变招形态**:**仅当怪物处于终结态(HP ≤ 20% maxHP)时**:
  - 伤害 ×0.7 × 2 = **净 ×1.4 倍**
  - 终结属性正常结算(可造成完美终结奖励)
- **非终结态下**:变招按钮灰显(条件不满足),即使有 VP 也不能用
- **适用场景**:对残血 BOSS / 精英的最后一击收割
- **取舍**:伤害 ×1.4 仅在终结态生效,平时(HP > 20%)完全无法用——这是设计强约束,玩家必须等

### 4.5 `variation_momentum` 变招·蓄势

- **普通形态**:原伤害
- **变招形态**:伤害 ×0.7, **本次出牌消耗的资源减半**(下限 1;非金钱消耗类)
  - MP 消耗卡:消耗 `ceil(原消耗 / 2)`,下限 1 MP
  - HP 消耗卡:消耗 `ceil(原消耗 / 2)`,下限 1 HP
  - 金币 / 技能 / 影响力消耗卡:**不享受此加成**(本设计仅针对 HP / MP)
- **适用场景**:高消耗卡(战斧猛劈 HP 15、坚守阵地 MP 12、清理地城 HP+MP 各 6)
- **取舍**:伤害减 30%,换来"消耗减半"——对资源紧张的中后期战斗价值高

## 5. 4 个追击类标签(保持不变)

追击类保持原状,**不受本轮重构影响**。简列参考:

| ID | 名称 | 触发器 | 效果 |
|----|------|--------|------|
| `pursuit_strike` | 追击·连携 | `dupe_2` | 手牌同名 ≥2 张(不含本张),打出追加 50% 伤害 |
| `pursuit_bleed` | 追击·流血 | `dupe_2` | 手牌同名 ≥2 张,追加 50% + 目标流血 3 回合 |
| `pursuit_armor_break` | 追击·破甲 | `dupe_2` | 手牌同名 ≥2 张,追加 50% + 破甲(下次受击护盾 −25%) |
| `pursuit_echo` | 追击·回响 | `per_use` | 本张牌结算后立即免费追加 1 次(不消耗行动点) |

字段定义保持:
```js
{
  id: 'pursuit_strike',
  name: '追击·连携',
  kind: 'pursuit',
  trigger: 'dupe_2',           // 保留
  effect: '手牌同名 ≥2 张,打出追加 50% 伤害',
  color: '#4a7a8a',
  apply: ctx => ({ damageBonus: 0.5 })
}
```

> 详见 [01-卡牌系统.md §6.4](./01-卡牌系统.md) 旧版本说明 + 历史 git log。本轮文档聚焦变招,追击代码与字段定义不在改动范围。

## 6. 战斗管线嵌入点

### 6.1 玩家攻击管线(02 §3.2)新阶段 2.5

```
阶段 1: 投伤害骰(damageRoll = 1d6,diceMul = 0.5x - 3x)
阶段 2: 基础伤害段(baseDamage = card.baseDamage × skillBonus × diceMul)
阶段 2.5 (新): 变招形态处理
   if (本次出牌为变招形态 && card.upgrades 含 variation_*):
     baseDamage = floor(baseDamage × 0.7)
     // 应用变招特殊效果(穿透 / 追加段 / 群攻 / 终结×2 / 蓄势减半)
阶段 3: 属性倍率(attributeMul = 3.0 / 1.0 / 0.3,取卡牌属性中最高)
阶段 4: 终结态强制置 0(若 !cardHasFinisherAttr(card))
阶段 5: 暴击(achievement /20 概率,×2)
阶段 6: 追击追加(若手牌同名 ≥2,pursuitDamage = ...)
阶段 7: 怪物护盾吸收
```

### 6.2 关键约束

- **×0.7 应用时机**:阶段 2.5,在阶段 3(属性倍率)之前
  - 理由:让属性倍率、暴击、技能 buff 都吃满变招的主段(只是底数被压 30%)
- **追加段**(`variation_chain`):独立结算阶段 3~5,不受 ×0.7 影响
- **护盾穿透**(`variation_pierce`):阶段 7 直接 `shieldAbsorb = 0`
- **群攻**(`variation_cleave`):遍历 `gameState.allEnemies`(当前 demo 通常 1 个)
- **终结态 ×2**(`variation_execute`):阶段 2.5 末附加 `× 2 if (monster.isFinisherState)`

### 6.3 与追击的优先级

- **变招 + 追击 同时存在**:变招处理在阶段 2.5(压主段 30%),追击处理在阶段 6(独立追加段)
- **追加段 vs 追击追加**:
  - `variation_chain` 追加段:走完阶段 2.5 → 3 → 4 → 5 的完整链路(享受属性 / 暴击)
  - `pursuit_*` 追击追加:阶段 6 单独计算 `card.baseDamage × skillBonus × diceMul × 0.5 × attributeMul`
  - 两者独立,可叠加(主段 + chain 追加段 + pursuit 追加段 = 三段)

## 7. 卡牌升级事件池(基本不变)

### 7.1 4 个事件

> **重要变化**:事件机制仍负责「卡牌挂上变招 / 追击标签」(永久);变招的「激活方式」已改为手动。**事件本身不消耗 VP**——VP 是战斗内资源。

| 事件 ID | 名称 | 候选池 | 玩家可选 | weight | 金币成本 |
|---------|------|--------|---------|--------|----------|
| `knight_mentor` | 老骑士的指点 | 穿刺 / 连携 / 破甲 | 选 1 个 | 1.0 | 0 |
| `ancient_scroll` | 古武残卷 | 连斩 / 连携 | 选 1 个 | 1.0 | 0 |
| `black_market` | 黑市商人 | 流血 / 破甲 / 终结 | 选 1 个 | 1.0 | 1000 |
| `secret_vault` | 宗门密藏 | 裂地 / 终结 | 选 1 个 | 0.5(稀有) | 0 |

事件池按 `weight` 抽取:
- **击败怪物** 30% 概率触发随机事件,事件分类中 `cardUpgrade` 占 10% 权重
- **事件节点** 100% 触发,事件分类同 5 类分布
- `secret_vault` 权重 0.5 → 出现概率为其他事件的 1/2

### 7.2 二段式选择弹窗(基本沿用旧版)

```
[触发卡牌升级事件]
  ↓
[Modal #card-upgrade-modal 打开]
  ├─ 第一段:候选升级池 → 玩家选 1 个 _pendingUpgrade.pickedTagId
  │
  └─ 第二段:当前手牌中所有"动作牌"(attack / defense / function)
              → 玩家选 1 张
              → 边界:手牌无动作牌 + 卡组有 → 随机挂到卡组 1 张
              → 边界:卡组也无动作牌 → 跳过 + 弹窗关闭
              → 边界:同一卡已挂该升级 → 阻止覆盖

确认 → attachUpgradeToCard(targetCard, pickedTagId)
  └─ targetCard.upgrades.push(pickedTagId)
  └─ battle-log: "📜 卡牌 [X] 习得 [Y]!"

[弹窗关闭 + continueAfterNonBattle 推进下一节点]
```

## 8. 边界处理

### 8.1 VP 用尽

- 变招按钮灰显
- tooltip: "⚡ VP 已耗尽,等待下回战斗"
- 不影响追击 / 普通形态出牌

### 8.2 变招 + 追击叠加

- 同一张卡可同时挂变招 + 追击标签
- 出牌时:
  - 选「普通形态」→ 仅追击生效(自动)
  - 选「变招形态」→ 变招生效(×0.7 + 特殊)+ 追击生效(自动追加)
- 追击追加段独立计算,不受 ×0.7 影响

### 8.3 同牌多次变招

- 一场战斗中,同一张牌可多次选「变招形态」(每次扣 1 VP)
- 每张卡独立计「本场变招次数」(可写入 `card.activationLog` 用于 UI 统计,但不影响机制)

### 8.4 跨战斗 VP 重置

- 每场战斗开始 `startBattle()` 重置 VP = 3
- 旧战斗未用完的 VP 不累积到下一场

### 8.5 变招·终结 在非终结态不可用

- `variation_execute` 变招按钮在非终结态下灰显
- 即使 VP > 0,tooltip 提示 "需进入终结态(敌人 HP ≤ 20%)"

## 9. 与已有系统的关系

### 9.1 vs 卡牌属性(01 §6)

```
卡牌属性   = 牌的固有字段(achievement / authority)
           → 决定对特定怪物的伤害倍率与终结资格
变招/追击 = 玩家后挂的标签(upgrades: [])
           → 变招:战斗中手动 + 消耗 VP + ×0.7 + 特殊效果
           → 追击:战斗中自动 + 无代价 + 纯增益
```

三者**正交**:一张"英勇冲锋"(成就)+ 挂"变招·连斩" + 挂"追击·连携" → 仍按成就属性走,变招主段 ×0.7 + 追加段,追击追加 50%。

### 9.2 vs 终结态(02 §2.2)

- 进入终结态:`isFinisherState = true`
- `variation_execute` 在终结态下启用,伤害 ×0.7 × 2 = 净 ×1.4
- `variation_execute` 仍需含终结属性才能造成完美终结奖励

### 9.3 vs 驿站强化(03 §7.2)

| 机制 | 入口 | 永久性 | 触发条件 | 代价 |
|------|------|--------|----------|------|
| **驿站强化** | 驿站节点 + 金币 | ✓(永久数值) | 永久 | 金币 |
| **变招** | 卡牌升级事件 + 战斗内 | ✓(标签永久)+ 战斗内手动激活 | 战斗节点 + VP > 0 | 伤害 ×0.7 + 1 VP |
| **追击** | 卡牌升级事件 | ✓(永久) | 触发器(自动) | 无 |

三轨互补:驿站给数值,变招给机制(代价型),追击给机制(白嫖型)。

### 9.4 vs 委托卡(work)

委托卡**不参与变招**。理由:委托卡的"动作"语义过于仪式化(HP+MP 双重消耗),强行挂变招会让"工作"流派过度膨胀,且与变招的"花 VP 换机制"模型语义不符。

委托卡同样不参与追击(沿用旧规则)。

### 9.5 vs 伴侣赐福(04 §4.1)

> 04 §4.1 「增益 / 升级 / 重抽类」提到「变招、追击额外 +1」。

- **追击 +1**:沿用旧语义——某些伴侣赐福使追击触发后"额外 +1 次",与变招 VP 系统正交
- **变招 +1**:在新语义下可解释为「每场战斗 VP 上限 +1」(如 VP 从 3 变 4),具体实现由 04 文档进一步细化
