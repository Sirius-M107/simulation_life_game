## 1. 怪物体系

怪物分为普通怪、精英怪、BOSS 怪，随楼层提升属性变强，对应各种奇幻生物。

### 1.1 普通怪物（无属性交互）

> 普通怪**不涉及属性抗性/终结态**，攻击牌将血量降至 0 即可击杀。

| 名称 | HP | 伤害 | 特点 |
|------|-----|------|------|
| 哥布林掠夺者 | 低 | 低 | 喜欢偷窃，常说"这年头生意难做" |
| 史莱姆沼泽妖 | 低 | 低 | 自称"沼泽诗人"，只会吐泡泡 |
| 森林地精 | 中 | 中 | 藏在草丛中突袭 |
| 荒野野狼 | 低 | 低 | 成群结队，降低玩家收益 |

### 1.2 精英怪物（每个独立配置属性交互）

|------|-----|------|----------|

| 黑暗法师 | 中 | 中 | 削弱护盾，心灵压制 |
| 狼人 | 中 | 高频小额 | 群攻特性 |
| 食人魔 | 很高 | 递增 | 伤害随回合递增，越拖越强 |

## 2. 怪物属性配置（每个独立设计）

### 2.1 配置字段

```js
{
  name: '...',
  hp: 60,
  damage: 15,
  skill: 4,
  gold: 400,
  exp: 5,
  // 属性抗性 / 加伤配置（自定义多属性组合）
  attributes: {
    achievement: { damageMul: 2.0, type: 'weak' },    // +200% 伤害（弱点）
    authority:   { damageMul: 0.3, type: 'resist' },  // -70% 伤害（抗性）
    neutral:     { damageMul: 1.0, type: 'normal' }   // 100% 伤害
  },
  // 终结属性
  finisherAttribute: 'achievement',  // 必杀属性：怪物 HP ≤ 终阈时只有含此属性的牌能造成最后伤害
  finisherThreshold: 0.2             // HP 比例（≤20% 时进入终结态）
}
```

### 2.2 终结态机制

- **触发条件**：怪物 HP / maxHP ≤ `finisherThreshold` → 进入**终结态**。
- **终结态下**：
  - 含终结属性的牌 → 正常结算伤害，享受属性加伤。
  - 不含终结属性的牌 → **伤害强制置 0**（无法扣血，但非致命段可正常结算）。
  - **变招 / 追击追加的伤害**也必须含终结属性，否则同样置 0。
- **完美终结**：
  - 终结态被含终结属性的牌击杀 → 触发"完美终结"奖励。
  - 奖励内容：金币 / 经验 / 影响力 +20%。
  - 写入 `gameState.perfectFinishers[]` 用于成就系统。

### 2.3 抗性 / 加伤计算顺序

> 本节保留为旧公式的速查参考，**当前权威定义见 [§3.2 玩家攻击管线](#32-玩家攻击管线对怪物造成伤害)**。

```
finalDamage = (baseDamage + variationBonus) * attributeMul + pursuitBonus
```

- `baseDamage`：卡牌基础伤害
- `variationBonus`：变招 tier 提升的伤害加成
- `attributeMul`：属性倍率（取卡牌属性中最高，不叠加）
- `pursuitBonus`：追击追加伤害（独立计算 attributeMul）

新管线（[§3.2](#32-玩家攻击管线对怪物造成伤害)）在此基础上新增：
- 阶段 1：`damageRoll = rollD6()` 作为伤害倍率（0.5x - 3x）
- 阶段 2：技能加成 `skillBonus = 1 + skill/200` 合并到 `baseDamage`
- 阶段 6：暴击双倍（应用在 baseDamage 与 pursuitDamage 两段）

### 2.4 破局机制（防卡死）

> 当玩家手牌中**所有攻击牌都不含终结属性**时，可能陷入无法终结的僵局。提供以下破局机制：

- **属性印记（消耗品）**：
  - 商店出售，价格 300 金币。
  - 使用后为 1 张攻击牌添加临时终结属性（**仅 1 场战斗有效**）。
- **重新洗牌（驿站功能）**：
  - 驿站多功能节点提供。
  - 消耗 50 MP，重抽 5 张手牌。

## 3. 战斗系统

### 3.1 核心机制：双方独立攻击回合（D&D 风格）

> **核心改动**：每回合双方都攻击，骰子只决定伤害倍率，不再决定攻击权（取消"差值×2"机制）。

**回合流程**：

```
回合开始
  ├── 玩家攻击阶段：出牌 → 投伤害骰 (1d6) → 8 阶段管线结算对怪物伤害
  └── 怪物攻击阶段：投伤害骰 (1d6) → 3 阶段管线结算对玩家伤害
回合结束（弃剩余手牌）
```

**骰子角色重新定位**：
- 旧设计：1d6 作为攻击点数，用于决定攻击权 + 计算差值×2。
- 新设计：1d6 作为**伤害倍率**（0.5x - 3x），即 `diceMul = roll × 0.5`。

### 3.2 玩家攻击管线（对怪物造成伤害）

**8 阶段结算**：

```javascript
function playerAttackDamage(card, player, monster) {
  // 阶段 1: 投伤害骰（D&D 伤害骰 1d6）
  const damageRoll = rollD6();           // 1-6
  const diceMul = damageRoll * 0.5;     // 映射到 0.5x - 3x 倍率

  // 阶段 2: 基础伤害段（技能加成 + 骰子倍率）
  const skillBonus = 1 + player.skill / 200;   // 每 10 技能 +5%
  let baseDamage = card.baseDamage * skillBonus * diceMul;

  // 阶段 2.5: 变招形态处理（玩家手动启用,消耗 1 VP,见 08-变招.md）
  //   若本次出牌为变招形态,主段伤害 ×0.7,并应用变招特殊效果:
  //     - variation_pierce : ignoreShield=true（阶段 7 shieldAbsorb=0）
  //     - variation_chain  : 追加一段 card.baseDamage * 0.5 * attributeMul（独立结算）
  //     - variation_cleave : 改为群攻所有敌人
  //     - variation_execute: 若 monster.isFinisherState,×2(净 ×1.4)
  //     - variation_momentum: costReduction=0.5（仅 HP/MP 消耗,金币不减）
  if (player._thisPlayIsVariation && hasVariation(card)) {
    baseDamage = Math.floor(baseDamage * 0.7);
    // 特殊效果在阶段 3-7 各处读取 variation_id 分支处理
  }

  // 阶段 3: 属性倍率（D&D 抗性 / 弱点）
  const attributeMul = getAttributeMul(card, monster);   // 3.0 / 1.0 / 0.3
  baseDamage *= attributeMul;

  // 阶段 4: 终结态强制置 0（精英 / BOSS 专属）
  if (monster.isFinisherState && !cardHasFinisherAttr(card, monster)) {
    baseDamage = 0;
  }

  // 阶段 5: 暴击（成就感 D&D 自然 20 = 双倍）
  const isCrit = Math.random() < player.achievement / 20;
  if (isCrit) baseDamage *= 2;

  // 阶段 6: 追击追加（D&D 双重伤害独立计算 attributeMul）
  //   注意:追击追加独立计算 baseDamage,不参与阶段 2.5 的 ×0.7
  let pursuitDamage = 0;
  if (hasPursuit(card) && handHasSameNameNotSelf(card, player.hand) >= 2) {
    // +50% base × 技能 × 骰子倍率 × attributeMul，独立 attributeMul
    pursuitDamage = card.baseDamage * skillBonus * diceMul * 0.5 * attributeMul;
    if (isCrit) pursuitDamage *= 2;
  }

  // 阶段 7: 怪物护盾吸收（D&D 护甲先于生命）
  //   变招·穿刺:若 player._thisPlayIsVariation && variation=穿刺,则 shieldAbsorb = 0
  let totalDamage = baseDamage + pursuitDamage;
  const shieldAbsorb = Math.min(monster.shield, totalDamage);
  totalDamage -= shieldAbsorb;
  monster.shield -= shieldAbsorb;

  return totalDamage;
}
```

### 3.3 怪物攻击管线（对玩家造成伤害）

**3 阶段结算**：

```javascript
function monsterAttackDamage(monster, player) {
  // 阶段 1: 投伤害骰
  const damageRoll = rollD6();
  const diceMul = damageRoll * 0.5;

  // 阶段 2: 基础伤害段
  let damage = monster.baseDamage * diceMul;

  // 阶段 3: 影响力百分比减伤（influence / 15 %，上限 50%）
  const reductionPct = Math.min(0.5, player.authority / 15 / 100);
  damage *= (1 - reductionPct);

  // 阶段 4: 玩家护盾吸收
  const shieldAbsorb = Math.min(player.shield, damage);
  damage -= shieldAbsorb;
  player.shield -= shieldAbsorb;

  return damage;
}
```

### 3.4 特殊机制叠加

- **暴击**（成就感 /20 概率）：在阶段 6 应用，**双倍**基础伤害段与追击段（D&D 自然 20 风格）
- **变招减伤**（阶段 2.5,见 [08-变招.md](./08-变招.md)）：玩家在战斗中手动启用变招时,主段伤害统一 ×0.7;附加特殊效果（穿刺/连斩/裂地/终结/蓄势）在后续阶段生效。仅战斗节点可用,每场战斗上限 3 次（消耗 VP）
- **伤害倍率**（技能 /10 点 +5%）：在阶段 2 应用，合并为 `skillBonus = 1 + skill/200`
- **威吓减伤**（影响力 /15 %，上限 50%）：在怪物攻击管线阶段 3 应用，**百分比**减伤
- **护盾**：先扣护盾，护盾耗尽后再扣体力（D&D 临时生命 / Absorb shield 风格）

> **注意**：原 §3.3 "属性伤害计算" 章节描述的旧公式（`finalDamage = (baseDamage + variationBonus) * attributeMul + pursuitBonus`）已被本节 8 阶段管线取代，详细流程见 [§3.2 玩家攻击管线](#32-玩家攻击管线对怪物造成伤害) 与 [§3.3 怪物攻击管线](#33-怪物攻击管线对玩家造成伤害)。

## 4. 战斗状态管理

### 4.1 gameState 战斗相关字段

```js
gameState = {
  // ... 资源字段
  cardUsageCount: {},          // 本场战斗内每张基础卡的累计使用次数（保留,变招不再依赖,但仍可用于 UI 统计）
  variationPoints: 3,          // 变招点（VP）— 每场战斗开局 3,全局共享池;startBattle() 重置
  perfectFinishers: [],        // 完美终结记录（成就系统用）
  currentEnemy: null,          // 当前怪物实例
  _thisPlayIsVariation: false, // 本次出牌是否启用变招（executeCardEffect 读取后重置）
  // ... 其他战斗状态
};
```

### 4.2 战斗开始 / 结束

- **战斗开始**（`startBattle`）：清空 `cardUsageCount = {}`,初始化 `variationPoints = 3`。
- **战斗结束 / 换层**：清空 `cardUsageCount = {}`,`variationPoints` 由下次 startBattle 重置。

> **变招 `usageCount` 范围**：本场战斗内累计，跨战斗清零。
> **变招点（VP）范围**：每场战斗开局固定 3 点,跨战斗重置;不写入存档。详见 [08-变招.md](./08-变招.md)。
> **升级标签** 永久跟随卡牌实例（不依赖战斗状态，跨战斗保持）。
