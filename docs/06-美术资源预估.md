> 本文档基于现有设计文档（00-05）估算游戏所需的美术资源量级，指导 ComfyUI 批量产出与资源管理。

## 0. 美术风格基线

- **整体调性**：中世纪油画风格（暖色调 + 画布纹理）
- **运行环境**：纯前端网页游戏（`index.html` + `style.css` + `script.js`，无构建步骤）
- **生产工具**：ComfyUI（Windows 11）
- **资源路径约束**：所有路径必须**相对**，不依赖绝对路径（保证 WSL ↔ Windows 兼容）
- **格式建议**：WebP（而非 PNG），首屏懒加载

## 3. 阶段化产出建议

若单人 + ComfyUI，建议分阶段产出，避免一开始画太多。

### 🥇 第一阶段：MVP（可玩）

- 20 张卡牌插画（最小集）
- 17 个怪物主图（暂不做终结态变体）
- 主角 + 5 位伴侣各 1 张肖像（暂不做 3 阶段）
- 资源 / 属性 / 状态 / 地图节点 共 ~24 个图标
- 1 张塔背景（先不做分层）
- **总计：约 65 张，约 15-20 MB**

### 🥈 第二阶段：完整版

- 补全 BOSS 终结态（+9 张）
- 补全伴侣 3 阶段（+10 张）
- 补全 3 层背景 + 节点变体
- **总计：约 95 张**

### 🥉 第三阶段：打磨

- 卡牌特效、UI 动效贴图、商店 / 驿站细节图
- BOSS 入场动画、终结特效

## 4. 技术轻量化建议

| 技巧 | 效果 |
|------|------|
| 输出 **WebP**（不用 PNG） | 体积直接减 30-50% |
| **懒加载**：进入下一层时再加载该层素材 | 首屏只需 ~5 MB |
| 用 **CSS Sprite / 图集** 合并小图标 | 减少 HTTP 请求 |
| 怪物用 **缩略图 + 点击放大** | UI 区用 200×200 即可 |

25-35 MB 在 WSL 代码仓库里 **完全用 Git LFS 即可**，无需云存储。

## 6. 资源引用规范（与代码对接）

代码中**只用稳定的 ID** 引用资源，便于后续替换美术：

```js
const ASSET_PATHS = {
  cards: {
    atk1: 'assets/cards/attack/charge.webp',
    atk2: 'assets/cards/attack/sword-slash.webp',
    // ...
  },
  monsters: {
    'goblin-raider':    'assets/monsters/common/goblin.webp',
    'ancient-dragon':   'assets/monsters/boss/dragon.webp',
    'ancient-dragon-x': 'assets/monsters/boss/dragon-finisher.webp',  // 终结态
  },
  partners: {
    isabella: 'assets/partners/isabella/stage1.webp',
    // ...
  }
};

// 加载示例
const cardImg = new Image();
cardImg.src = ASSET_PATHS.cards.atk1;
```

> 命名规则：**英文 ID + 横线连接**，不用中文路径（避免编码问题）。
