## 1. 怪物体系

怪物分为普通怪、精英怪、BOSS 怪，随楼层提升属性变强，对应各种奇幻生物。

### 1.1 普通怪物（无属性交互）

> 普通怪**不涉及属性抗性/终结态**，攻击牌将血量降至 0 即可击杀。

| 名称 | HP | 伤害 | 特点 |
|------|-----|------|------|
| 哥布林掠夺者 | 低 | 低 | 喜欢偷窃，常说"这年头生意难做" |
| 史莱姆沼泽妖 | 低 | 低 | 自称"沼泽诗人"，只会吐泡泡 |
| 森林地精 | 中 | 中 | 藏在草丛中突袭 |
| 荒野野狼 | 低 | 低 | 成群结队，降低玩家收益 |

### 1.2 精英怪物（每个独立配置属性交互）

| 名称 | HP | 伤害 | 特殊机制 |
|------|-----|------|----------|
| 堕落骑士 | 高 | 高 | 打乱玩家出牌节奏 |
| 黑暗法师 | 中 | 中 | 削弱护盾，心灵压制 |
| 狼人 | 中 | 高频小额 | 群攻特性 |
| 食人魔 | 很高 | 递增 | 伤害随回合递增，越拖越强 |

> **属性配置示例**（每个精英独立设计）：
> - 堕落骑士：成就感 +200%（weak）、影响力 -70%（resist）、终结属性 = 成就感
> - 黑暗法师：影响力 +200%（weak）、成就感 -70%（resist）、终结属性 = 影响力
> - 狼人：双属性 +150%（weak）、终结属性 = 成就感
> - 食人魔：影响力 +200%（weak）、成就感 -50%（resist）、终结属性 = 成就感

## 2. 怪物属性配置（每个独立设计）

### 2.1 配置字段

```js
{
  name: '...',
  hp: 60,
  damage: 15,
  skill: 4,
  gold: 400,
  exp: 5,
  // 属性抗性 / 加伤配置（自定义多属性组合）
  attributes: {
    achievement: { damageMul: 2.0, type: 'weak' },    // +200% 伤害（弱点）
    authority:   { damageMul: 0.3, type: 'resist' },  // -70% 伤害（抗性）
    neutral:     { damageMul: 1.0, type: 'normal' }   // 100% 伤害
  },
  // 终结属性
  finisherAttribute: 'achievement',  // 必杀属性：怪物 HP ≤ 终阈时只有含此属性的牌能造成最后伤害
  finisherThreshold: 0.2             // HP 比例（≤20% 时进入终结态）
}
```

### 2.2 终结态机制

- **触发条件**：怪物 HP / maxHP ≤ `finisherThreshold` → 进入**终结态**。
- **终结态下**：
  - 含终结属性的牌 → 正常结算伤害，享受属性加伤。
  - 不含终结属性的牌 → **伤害强制置 0**（无法扣血，但非致命段可正常结算）。
  - **变招 / 追击追加的伤害**也必须含终结属性，否则同样置 0。
- **完美终结**：
  - 终结态被含终结属性的牌击杀 → 触发"完美终结"奖励。
  - 奖励内容：金币 / 经验 / 影响力 +20%。
  - 写入 `gameState.perfectFinishers[]` 用于成就系统。

### 2.3 抗性 / 加伤计算顺序

```
finalDamage = (baseDamage + variationBonus) * attributeMul + pursuitBonus
```

- `baseDamage`：卡牌基础伤害
- `variationBonus`：变招 tier 提升的伤害加成
- `attributeMul`：属性倍率（取卡牌属性中最高，不叠加）
- `pursuitBonus`：追击追加伤害（独立计算 attributeMul）

### 2.4 破局机制（防卡死）

> 当玩家手牌中**所有攻击牌都不含终结属性**时，可能陷入无法终结的僵局。提供以下破局机制：

- **属性印记（消耗品）**：
  - 商店出售，价格 300 金币。
  - 使用后为 1 张攻击牌添加临时终结属性（**仅 1 场战斗有效**）。
- **重新洗牌（驿站功能）**：
  - 驿站多功能节点提供。
  - 消耗 50 MP，重抽 5 张手牌。

## 3. 战斗系统

### 3.1 核心机制：点数对比 + 多维加成

**回合流程**：
1. 玩家出卡 → 消耗 HP/MP/金币/技能/影响力
2. 系统生成玩家骰子点数 = 基础骰子(1-6) + 技能加成(skill/5)
3. 系统生成敌人骰子点数 = 基础骰子(1-6)
4. 对比差值：
   - **玩家点数 > 敌人点数**：对敌人造成（基础伤害 + 差值×2）的伤害
   - **敌人点数 > 玩家点数**：玩家受到（差值×2 - 影响力减伤）的伤害
   - **平局**：无事发生

### 3.2 特殊机制叠加

- **暴击**（成就感 /20 概率）：造成双倍伤害
- **伤害倍率**（技能 /10 点 +5%）：在基础伤害上叠加
- **威吓减伤**（影响力 /15）：减少敌人造成的伤害
- **护盾**：先扣护盾，护盾耗尽后再扣体力

### 3.3 属性伤害计算（含变招/追击/终结态）

**伤害公式**：
```
finalDamage = (baseDamage + variationBonus) * attributeMul + pursuitBonus
```

#### 变招触发
- 出牌时扫描卡牌 `upgrades` 数组中的 `variation_*` 标签。
- 检查 `gameState.cardUsageCount[card.id]`（本场战斗累计使用次数）。
- 满足触发条件 → 调整 `baseDamage` + 应用变招效果（穿透/连斩/群攻/终结）。
- 详细触发器见 [03-事件与驿站.md §6.2](./03-事件与驿站.md)。

#### 追击触发
- 出牌时扫描当前手牌中同名卡牌数量（不含本张）。
- 满足触发条件 → 追加 50% 伤害（**不重投骰子**、**不消耗行动点**）。
- 追击伤害**独立计算 attributeMul**，避免双倍叠加。
- 终结态下追加的追击伤害也受终结属性限制。

#### 终结态判定
- 怪物 HP / maxHP ≤ `finisherThreshold` → 进入终结态。
- 含终结属性 → 正常结算。
- 不含终结属性 → 伤害置 0。
- 完美终结（终结态 + 含终结属性击杀）→ 奖励 +20%。

#### 完整出牌结算流程
1. **资源消耗**：`consumeResource(card)` 按 `cost.type` 路由（HP/MP/金币/技能/影响力）。
2. **变招判定**：扫描 `variation_*` 标签 → 调整 `baseDamage`。
3. **属性倍率计算**：从怪物 `attributes` 中匹配卡牌属性，取最高 `damageMul`。
4. **骰子对比**：跑原有逻辑，得到基础伤害段。
5. **终结态检查**（仅精英/BOSS）：
   - 怪物 HP 进入终结区间 + 本牌不含 `finisherAttribute` → 伤害置 0 + 战斗日志提示。
6. **追击判定**：扫描手牌中同名卡牌 → 追加伤害（独立计算 attributeMul）。
7. **伤害结算**：`finalDamage = (base + varBonus) * attributeMul + pursuitBonus`。
8. **使用计数 +1**：`cardUsageCount[card.id]++`。

## 4. 战斗状态管理

### 4.1 gameState 战斗相关字段

```js
gameState = {
  // ... 资源字段
  cardUsageCount: {},          // 本场战斗内每张基础卡的累计使用次数
  perfectFinishers: [],        // 完美终结记录（成就系统用）
  currentEnemy: null,          // 当前怪物实例
  // ... 其他战斗状态
};
```

### 4.2 战斗开始 / 结束

- **战斗开始**（`startBattle`）：清空 `cardUsageCount = {}`。
- **战斗结束 / 换层**：清空 `cardUsageCount = {}`。

> **变招 `usageCount` 范围**：本场战斗内累计，跨战斗清零。
> **升级标签** 永久跟随卡牌实例（不依赖战斗状态，跨战斗保持）。
